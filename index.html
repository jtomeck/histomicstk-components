Components Setup

Tailwind Config

The tailwind config should be included in the <head> of your website. The Tailwind Play CDN is included here as well because needs to be included before the config. Nothing needs to be edited since the colors are defined as variables in the CSS, which is where you will input your own hex color values.
{% include tailwind-config.html %}

                

                    
                

            


            

                

                    
Styles

                    These styles should be included in the <head> as well after the tailwind config. As mentioned, the color variables --primary, --secondary and --accent are defined here. Input your own colors and your theming work is done.
                

                
{% include tailwind-styles.html %}

                

                

                    
                

            


            

                

                    
Automatic Color Generation

                    This script does a bunch of magic that takes the hex values you input into the css color variables, converts them to HSL (hue, saturation, lightness) colors and automatically calculates legible content colors, hover states, etc. It should be included before the closing </body> tag of your website. It shouldn't be needed, but if there is any strange color loading behavior you can defer the script like so: <script type="text/javascript" defer>
                

                
{% include colorgen-script.html %}
